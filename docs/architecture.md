# Framework Architecture

The Automated SQL Server Recovery Validation Framework operates as a deterministic restore validation pipeline.

It integrates with an existing backup ecosystem and executes automated recovery validation through four main components.


| Role | Procedure | Objective |
|------|-------------|-------------------|
| Planner | `[cfg].[usp_GetLatestBackupFiles]` | Determines the deterministic restore chain required to execute a recovery scenario. |
| Engine | `[cfg].[usp_RestorePointInTime]` | Executes the restore process using the restore chain generated by the planner. |
| Validator | `[cfg].[usp_ValidatePitrCanary]` | Validates logical correctness of the restore using deterministic canary records. |
| Orchestrator | `[cfg].[usp_RunRestoreTests]` | Coordinates automated restore validation scenarios. |

Each component work as a layer that covers each part of the process.
